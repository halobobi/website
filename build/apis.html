Time-based One-time Password Generator (TOTP)

Link to API: https://api.andrasbiro.work/totp

Parameters

Name Description Value type Minimum value Maximum value Default value
strLength(?) Defines how many characters long the returned keys are. Integer 6 16384 10
expTime(?) Defines in minutes how long until the returned keys expire. Integer / Floating-point 1 10080 4
arraySize(?) Defines how many keys are to be returned. Integer 1 15 1
validKey(?) Request the validation of provided key. String - - -

Usage and returned values

Stringified JSON objects are returned.

If validKey parameter is used, all other parameters are disregarded.

If any of the valid parameters are used, but a value is not provided an error message is returned.

All parameters can be omitted. If valid, the provided values are used, otherwise default values are used.

Examples:

Request Yield Description
.../totp {"response": {"validKeys": ["tyP7nDMJYX"]}} Default values are used, an array of one key with a length of 10 and a 4 minute expiration time is returned.
.../totp? strLength=6 {"response": {"validKeys": ["G0jszD"]}} An array of one key with a length of 6 and a 4 minute expiration time is returned.
.../totp? strLength=8& arraySize=3 {"response": {"validKeys": ["UbAvrMoH", "d3uyooHU", "51UXyVAe"]}} An array of three keys with a length of 8 and a 4 minute expiration time is returned.
.../totp? strLength=12& arraySize=2& expTime=2 {"response": {"validKeys": ["BwV59O7aT1VJ", "e6Qu935035k1"]}} An array of two keys with a length of 12 and a 2 minute expiration time is returned.
.../totp? validKey= notAValidKey {"response": {"isValid": false}} A false boolean value is returned, signaling that the requested key either expired or never existed.
.../totp? validKey= aValidKey {"response": {"isValid": true, "createTime": 1706914644012}} A true boolean value comfirming the validity and a timestamp of the key's creation time is returned.
